## 2. Data study — "The State of Interview Cheating 2026"  *(target #17)*

**Why it's the single highest-leverage asset:** original data becomes *the citable source*. One quotable stat ("__% of remote interviews now show AI-assist signals") gets linked hundreds of times, quoted by journalists, and — critically for GEO — **repeated verbatim by ChatGPT/Perplexity/AI Overviews**. It's SEO + AEO + PR + sales enablement in one page, and it's the thing Fabric/Sherlock/Honorlock *don't* own yet.

### Data inputs
1. **Trueyy's own aggregate detection data** (the moat) — anonymized, min. sample size stated honestly. e.g. % of monitored interviews flagged, frequency of each signal (paste velocity, app-switch, timing), tool-fingerprint distribution.
2. **Original survey** — 200–500 hiring managers / recruiters (LinkedIn + email list + a panel like Prolific/Pollfish). Ask: suspicion rate, tools seen, policies, detection confidence, cost of a bad hire.
3. **Curated secondary stats** (cited) to frame the landscape.

### Report structure
1. **Key findings** — 5–7 punchy, declarative, *quotable* stats in a "steal these stats" box up top. Designed for LLM extraction + press headlines.
2. **Methodology** — sample sizes, dates, how each signal is defined. (Credibility = citations.)
3. **The surge** — prevalence + trend since 2023 (GPT-4 era).
4. **The toolkit** — which tools dominate: Cluely, Interview Coder, ChatGPT, Final Round AI + share.
5. **Where it happens** — coding vs behavioral, seniority, remote vs onsite, geography.
6. **The tells** — signal breakdown (paste, app-focus, timing, gaze) — subtle product tie-in.
7. **Are teams catching it?** — detection rate, confidence, false-positive fears.
8. **The cost** — bad-hire cost, offers extended to cheaters.
9. **What teams are doing** — policies, proctoring, live-monitoring adoption.
10. **2027 outlook** — predictions.
11. **"Cite this report"** — canonical stat list, embed graphics, methodology, contact.

### Formats & distribution
- **Ungated HTML report page** (don't gate the report — gating kills links; gate a PDF/deep-cut for leads if needed).
- Each key stat as a **branded shareable image** (social + press).
- **Embeddable charts** + `Dataset`/`Article` schema + a "cite this" block.
- Feeds the **press/HARO** pitches (#10, #11, #12).
- **Refresh annually** → "2027" edition becomes a recurring franchise that re-earns links every year.
- **GEO framing:** write each finding as a clean declarative sentence with stat + source, so AI engines quote it cleanly.

### Checklist
- [ ] Define 6–8 report questions + which Trueyy signals answer each
- [ ] Pull aggregate detection data (anonymized, state N)
- [ ] Field the hiring-manager survey (target n ≥ 200)
- [ ] Write findings + design 6 stat graphics
- [ ] Build ungated report page + Dataset/Article schema + "cite this" block
- [ ] Pitch press + HARO; email list; distribute on PH/Reddit/LinkedIn
- [ ] Set an annual-refresh reminder (→ 2027 edition)

## 5. AEO / GEO — rank inside the AI answers

**Goal:** be the source ChatGPT, Perplexity, Google AI Overviews, Gemini & Claude cite for "detect Cluely," "best AI cheating detection," etc. Overlaps SEO but has its own levers.

- [ ] **Original data (the §2 study)** — the #1 GEO asset; AI engines quote stats verbatim
- [ ] **Get on the sources LLMs retrieve from** — Reddit, G2, Capterra, comparison/roundup sites (this is why §1 doubles as GEO work)
- [ ] **vs / alternative pages (§4a)** — exactly what LLMs pull for "best/compare" prompts
- [ ] **Keep `llms.txt` current** — done, auto-updates from content
- [ ] **Entity consistency** — `sameAs`, identical brand name + description everywhere — done; maintain it
- [ ] **Declarative, sourced sentences** — LLMs quote clean, attributable claims over hedged prose
- [ ] **Unlinked brand mentions matter** — being *named* alongside "interview integrity" teaches the models, even without a link
- [ ] **Monthly AEO audit:** run target prompts in ChatGPT / Perplexity / Gemini ("how do I detect Cluely," "best AI interview cheating detection tools," "Trueyy vs Honorlock") → log whether/where Trueyy appears → fix the gap (usually: missing page or missing citation source)
